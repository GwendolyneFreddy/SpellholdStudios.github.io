Présentation


Disponible pour :     BGII SoA, BGII ToB, Tutu, IWD HoW, IWD TotL, IWDII

Ce composant permet d'afficher correctement les armures et les robes lorsque les personnages de toutes les classes s'en équipent. Il contourne la limitation du moteur du jeu qui n'affiche pas les robes portées par les guerriers, prêtres et voleurs, ni les armures portées par les magiciens, en changeant l'animation des personnages selon les types d'armure dont ils sont équipés.

Consultez le site du mod ou son forum pour obtenir les dernières mises à jour.

1ppv4 : Changement d'armure encore amélioré de Galactygon


Comme son nom l'indique, il s'agit d'une version améliorée de ma première version améliorée du Changement d'armure de Galactygon (disponible dans le mod BGII Tweakpack).

En gros, ce composant modifie de manière dynamique les animations des personnages pour qu'elles correspondent aux armures qu'ils portent (puis, rétablit leur aspect initial quand ils les retirent). Il contourne la limitation du moteur du jeu qui n'affiche pas les robes portées par les guerriers, prêtres et voleurs, ni les armures portées par les magiciens, en modifiant l'animation des personnages selon les types d'armure dont ils sont équipés.

Si vous utilisez déjà le composant de BGII Tweakpack, vous devez le désinstaller avant d'installer celui-ci. Ce composant est conçu pour fonctionner avec les Avatars uniques de voleur, il n'est donc pas recommandé de manière autonome.

Grâce à des animations de voleurs fonctionnant correctement, cela évite l'utilisation d'une grande partie des scripts sur lesquels mon ancien correctif reposait, tout en offrant un contenu de qualité supérieure.



1ppv4 : Changement d'avatar pour IWD


Il s'agit d'une adaptation du composant de changement d'avatar pour IWD HoW/TotL. Pour autant que je sache, c'est le premier mod qui le fait.

Le moteur du jeu IWD impose d'implémenter différemment certaines parties du code. Si le moteur d'HoW prend en charge les fichiers .EFF, il ne leur permet pas de déclencher des sorts. Les sorts ne peuvent pas non plus être directement liés à un objet, car quels que soient leurs paramètres, ils semblent se déclencher en continu (lorsque vous ne souhaitez les utiliser qu'une seule fois). Pour aggraver les choses, IWD ne dispose pas de script principal qui tourne en permanence.

Le composant contourne ces problèmes en vous permettant de basculer manuellement d'une animation à l'autre à l'aide des capacités spéciales des objets.

Lorsque vous vous équipez un objet, votre avatar ne changera pas automatiquement. Au lieu de cela, l'objet sera doté d'une capacité spéciale (cliquez sur Capacités d'objet dans la barre de raccourcis). Si vous estimez nécessaire de changer l'apparence du personnage (par exemple en équipant un guerrier d'une robe), utilisez cette capacité spéciale pour modifier son animation.

Dès que vous l'aurez utilisée, une capacité innée apparaîtra qui vous permettra de réinitialiser l'animation de base de votre personnage. Cette capacité innée restera disponible indéfiniment jusqu'à ce qu'elle soit utilisé ou qu'une autre armure soit équipée. Vous pouvez « revêtir » ou « retirer » visuellement la même armure tant que votre personnage en est équipé.

Le procédé n'est pas aussi pratique que celui employé par le composant pour BGII, mais deux clics (environ cinq secondes) l'emportent incontestablement sur la sauvegarde de la partie, la fermeture du jeu et l'édition de votre type d'animation dans NI ou Dalekeeper chaque fois que vous vous équipez d'un type d'armure différent.



1ppv4 : Changement d'avatar pour IWDII


Il s'agit d'une adaptation du composant de changement d'avatar pour IWDII.

Comme le moteur de WDII prend en charge les sorts liés aux objets, ce composant se trouve donc à mi-chemin entre les composants pour BGII et pour IWD.

Les objets changent votre apparence automatiquement lorsque vous vous en équipez, mais faute de script persistant, votre avatar ne retrouvera pas son aspect « normal » lorsque vous l'ôterez (donc, si vous équipez un combattant avec une robe, puis que vous la lui enlevez, il se retrouvera coincé avec l'animation de magicien sans armure).

Pour que vous puissiez revenir en arrière, ce composant ajoute une capacité innée, fonctionnant exactement comme celles de l’installation pour IWD. De la même manière, exactement comme dans le composant IWD, vous n'avez pas besoin de l'utiliser si vous changez d'armure.
